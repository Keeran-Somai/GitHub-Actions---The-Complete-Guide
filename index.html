Glossary

  • Repository: A directory or storage space where your project lives.
  • Commit: A snapshot of your files at a point in time.
  • Branch: An independent line of development.
  • Merge: Combine changes from different branches.
  • Stash: Temporarily save changes not ready to commit.
  • Tag: A marker for a specific commit, often used for releases.
  • Remote: A version of your project hosted on the internet or network.
  • HEAD: The current commit your working directory points to.

Common Git Commands

Command Description
git initInitialize a new Git repository
git clone <url>Clone a remote repository
git statusShow the working directory status
git add <file>Stage changes for commit
git commit -m "message"Commit staged changes
git logView commit history
git branchList branches
git checkout <branch>Switch branches
git merge <branch>Merge a branch into current
git pullFetch and merge from remote
git pushPush changes to remote

Typical Workflow

  1. Make changes to your files.
  2. Stage changes: git add .
  3. Commit changes: git commit -m "Describe your changes"
  4. Push to remote: git push
Tip: Use git status often to see what‚Äôs going on!

Branching Strategies

  • Feature Branches: Create a branch for each new feature or bugfix.
  • Main/Development Branches: Keep main or master stable; use develop for ongoing work.
  • Pull Requests: Merge changes via pull requests for code review.
  • Release Branches: Prepare releases in dedicated branches.
Example: git checkout -b feature/my-new-feature

Undoing Changes

  • Unstage a file: git reset <file>
  • Discard changes: git checkout -- <file>
  • Amend last commit: git commit --amend
  • Recover lost commits: git reflog
Trick: Use git reflog to recover lost commits!

6. Using .gitignore

The .gitignore file tells Git which files or folders to ignore in a project. This is useful for excluding files like build output, logs, or sensitive information.

# Example .gitignore
node_modules/
dist/
*.log
.env
  • Create a file named .gitignore in your project root.
  • Add patterns for files/folders you want Git to ignore.
  • Common patterns: node_modules/, *.log, .env

8. Advanced Git

  • Interactive Rebase: git rebase -i HEAD~n (edit last n commits)
  • Stash changes: git stash (save changes temporarily)
  • Apply stash: git stash apply
  • Cherry-pick: git cherry-pick <commit> (apply a specific commit)
  • Squash commits: Use interactive rebase to squash multiple commits into one
  • Bisect: git bisect start (find the commit that introduced a bug)
  • Tagging: git tag v1.0.0 (create a tag)
  • Show remote URLs: git remote -v

9. Tips & Tricks

  • See what changed in last commit: git show
  • See changes before committing: git diff
  • See changes for staged files: git diff --cached
  • Remove untracked files: git clean -fd
  • Rename a branch: git branch -m new-name
  • Set upstream branch: git push -u origin branch-name
  • List all aliases: git config --get-regexp alias
Tip: Use git log --oneline --graph --all for a visual commit history!
