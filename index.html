4. Glossary

  • Repository: Project storage with version history
  • Commit: A snapshot of your files at a point in time
  • Branch: An independent line of development
  • Merge: Combine changes from different branches
  • Stash: Temporarily save changes not ready to commit
  • Tag: A marker for a specific commit
  • Remote: A version of your project hosted elsewhere
  • HEAD: The current commit your working directory points to

7. Using .gitignore

The .gitignore file tells Git which files or folders to ignore in a project.

  • Create a file named .gitignore in your project root.
  • Add patterns for files/folders you want Git to ignore.
  • Common patterns: node_modules/, *.log, .env

8. Advanced Workflows

  • Interactive Rebase: git rebase -i HEAD~n
  • Stash: git stash
  • Cherry-pick: git cherry-pick <commit>
  • Bisect: git bisect start
  • Tagging: git tag v1.0.0

9. Troubleshooting

  • Merge Conflicts: Resolve conflicts, then git add and git commit.
  • Detached HEAD: Checkout a branch or create a new one.
  • Deleted Branch: Recover with git reflog.
  • Untracked Files: Check your .gitignore patterns.

11. FAQ

  • Undo last commit? git reset --soft HEAD~1 or git reset --hard HEAD~1
  • Global ignore? git config --global core.excludesfile ~/.gitignore_global
  • Who changed a line? git blame <file>
  • Recover deleted branch? git reflog
